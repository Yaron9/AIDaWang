纯小白实战如何建立免费个人博客网站

创建时间: 2025年8月3日 21:35

引言

很多老师希望拥有免费的个人博客网站，今天我们就一起基于GitHub Pages+Hexo搭建一个免费的博客网站。Hexo 是一个快速、简洁，而且功能强大的博客框架。我们可以使用 Markdown 编写博客文章，然后 hexo 帮我们把 Markdown 文件渲染成静态 HTML 页面。因此 hexo 非常适合用来搭建技术类博客，以及项目文档和个人网站。

本人纯小白，所以这个教程也是面向和我一样的小白的，那我们接下来就一步一步的搭建吧。

https://hexo.io/zh-cn/

    • source/_post：存放博客文章， hexo 提供了一篇范例文章 hello-world

2.2.2 GitHub Pages 部署博客

  1. 把本地文件映射到Github仓库后，就可以把项目部署到Github Pages上了。这样就可以在网上访问我们的博客网站了。具体操作步骤：
    首先在 myblog 项目的 _config.yml 配置文件中，把 git 服务部署的配置加进去。

    在文件最后填写：

    repository：替换为你自己的， 在code→SSH 复制过来。

  2. 然后安装 hexo-deployer-git 这个插件，来简化部署到 GitHub Pages 的流程

  3. 这样配置完以后，我们就可以在本地写文章，写完以后通过 http://localhost:4000/ 预览文章效果并进行修改。修改满意后，就可以部署到 GitHub Pages 这样其他人就能访问我们的博客了。使用流程如下：

    1. 创建博客：hexo new post 文章标题,然后我们看到左边源码Source/_posts 文件夹中生成一个md文件。我们可以直接在里面编辑博客内容即可。

    b. 在本地预览效果， 激活本地hexo服务

c. 然后把文章发布到 GitHub Pages。

💡
  • hexo generate：生成渲染，简写 hexo -g
  • hexo deploy：部署到 GitHub Pages，简写 hexo -d
  • hexo server：启动本地预览，简写 hexo -s
  • hexo clean：清除缓存，简写 hexo -c

d. 发布完成后，我们回到仓库，点击上方标签⚙Settings→左边栏Pages→Build and deployment→Deploy from branch→branch选择main→root→save(保存）

然后可以看到：Your site is live at https://yaron9.github.io/AIDaWang/
